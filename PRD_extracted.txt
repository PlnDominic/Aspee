1. Executive Summary
Aspee Pharmaceuticals requires a comprehensive, integrated Stock and Inventory Management System to digitize and streamline its end-to-end supply chain, warehouse operations, distribution, sales, and financial accounting processes. This document defines the complete product requirements for a web-based platform built on Supabase (PostgreSQL backend + Auth + Realtime), Node.js (API layer), and Next.js (frontend framework).
The system will replace manual or fragmented processes across six core operational domains: Purchasing, Stores & Production, Route-based Distribution, Sales, Customer Management, and Accounting. It is designed for multi-user, role-based access and will serve purchasing managers, store managers, production managers, route/van sales teams, accountants, and executive management.
2. Problem Statement
Aspee Pharmaceuticals currently faces the following operational challenges:
Lack of real-time visibility into stock levels across warehouses and vans
Manual purchase order and supplier management creating errors and delays
No systematic tracking of goods from purchase → storage → production → dispatch → sale
Disconnected sales and invoicing from inventory, leading to stock discrepancies
Accounting processes (receipts, payments, expenses, payroll) handled in silos
No consolidated dashboard for management decision-making
Difficulty tracking petty cash, tax obligations, and salary disbursements
3. Goals & Objectives
3.1 Primary Goals
Provide full traceability of pharmaceutical products from supplier to customer
Automate purchase orders, stock receipts, and supplier management
Enable real-time inventory tracking across stores, production, and delivery vans
Digitize sales invoicing, receipting, and customer accounts
Centralize accounting: money receipts, payments, expenses, salaries, tax, petty cash
Deliver a management dashboard with actionable KPIs and reports

7. Functional Requirements by Module
7.1 Dashboard
The dashboard is the landing page for all authenticated users, showing role-filtered KPIs and alerts.
7.1.1 Features
Real-time stock summary: total SKUs, low-stock alerts, out-of-stock items
Financial snapshot: today's receipts, payments, outstanding invoices
Recent activity feed: last 10 transactions across modules
Van/Route status cards: each van's loaded stock value and today's sales
Purchase order status: pending, approved, partially received, complete
Top-selling products (last 7 / 30 days)
Expiry alerts: products expiring within 30 / 60 / 90 days
Overdue customer balances widget
7.2 Purchasing Module — Supplier Management
7.2.1 Supplier Master
Create, edit, deactivate supplier records
Fields: Name, Contact Person, Phone, Email, Address, Tax ID, Payment Terms, Bank Details
Supplier category (local, imported, contract manufacturer)
Attach supplier documents (registration certificates, licenses)
View full purchase history per supplier
7.2.2 Purchase Orders (PO)
Create PO: select supplier, add line items (product, quantity, unit price, discount)
PO approval workflow: Draft → Submitted → Approved → Sent to Supplier
Auto-generate PO number (e.g., PO-2026-00001)
Attach PO document (PDF export) and email to supplier directly
Track expected delivery date per PO
Partial delivery support: PO remains open until fully received
PO amendment with audit trail (versioning)
7.2.3 Goods Receipt Note (GRN)
Create GRN against open PO: scan/select PO, confirm quantities received
Record batch numbers, expiry dates, and manufacturing dates per item
Record discrepancies: received qty vs ordered qty
Auto-update stock levels in Stores upon GRN confirmation
Link supplier invoice to GRN for 3-way matching (PO → GRN → Invoice)
Return to Supplier (RTS) note creation from GRN
7.3 Stores Module
7.3.1 Product Master
Product fields: Name, SKU/Barcode, Category, Unit of Measure, Pack Size
Pharmaceutical-specific: Dosage Form, Strength, Registration Number
Reorder level, minimum stock, maximum stock configuration
FIFO / FEFO (First Expiry First Out) costing method
Product images and document attachments
7.3.2 Stock Management
Real-time stock balance per product per location (main store, secondary stores)
Stock movement log: every IN/OUT transaction with timestamp, reference, user
Stock transfer between locations (e.g., main store to production store)
Stock adjustment (with reason codes: damaged, expired, counted, correction)
Physical stocktake module: freeze stock, count, reconcile, post adjustments
Batch / Lot tracking with full traceability
Expiry date management with automated alerts
7.4 Production Manager Module
7.4.1 Bill of Materials (BOM)
Define BOM per finished product: list of raw materials and quantities
Support multi-level BOM (intermediates and finished goods)
Version control on BOM with effective dates
7.4.2 Production Orders
Create production order: select finished product, quantity to produce, BOM version
Statuses: Draft → Released → In Progress → Completed → Closed
Auto-calculate raw material requirements from BOM
Issue raw materials from store: deducts stock, creates consumption log
Record actual consumption vs planned (yield tracking)
Receive finished goods into store upon production completion
Production cost calculation (materials + labour overhead)
7.5 Routes & Van Sales Module
7.5.1 Van / Route Setup
Create van profiles: Van ID, registration plate, assigned driver, route area
Define route plans: list of customers per route with delivery sequence
Assign products and target quantities per route
7.5.2 Van Loading
Create van loading order: select van, products, quantities to load
Loading requires approval before stock is deducted from main store
Print loading list for warehouse staff
Van inventory balance tracked separately per van
7.5.3 Route Sales (Van Invoicing)
Sales rep creates invoices on the road (mobile-optimised PWA)
Select customer from route list, add products from van stock
Collect payment (cash) and record receipt on spot
Unsold stock returned to main store at end of day (Van Return)
End-of-day van reconciliation: loaded - sold - returned = 0
Offline mode: queue transactions when no internet, sync on reconnect
7.6 Sales Module
7.6.1 Sales Invoices
Create invoice: select customer, add products, apply discount, add tax
Invoice types: Cash Sale, Credit Sale, Proforma Invoice
Auto-generate invoice number (e.g., INV-2026-00001)
PDF export and email/WhatsApp send
Invoice statuses: Draft → Issued → Partially Paid → Paid → Overdue → Voided
Credit limit enforcement: block invoice if customer exceeds credit limit
Stock is reserved on invoice creation and deducted on confirmation
7.6.2 Credit Notes
Create credit note against original invoice (returns, price corrections)
Partial or full credit note support
Credit note reverses stock and adjusts customer balance
7.6.3 Sales Receipts
Record payment against invoice: Cash or Cheque
Cheque details: bank name, cheque number, date, amount
Partial payment support with running balance
Automatic ledger entry on receipt confirmation
7.7 Customer Management Module
Customer master: Name, Address, Contact, Tax ID, Route assignment
Credit limit and payment terms configuration
Customer Statement of Account (SOA): all invoices, receipts, credits, balance
Overdue analysis: 0-30 days, 31-60 days, 61-90 days, 90+ days
Customer price list / special pricing tiers
Transaction history: all invoices and payments
Credit hold: block new invoices for customers with overdue balances
7.8 Accounting Module
The accounting module implements double-entry bookkeeping integrated with all operational modules. Every financial event auto-generates the appropriate journal entries.
7.8.1 Chart of Accounts
Configurable chart of accounts: Assets, Liabilities, Equity, Revenue, Expenses
Account hierarchy: Class → Group → Sub-group → Account
Default account mapping for automated journal entries
7.8.2 Money Receipts (Debit Entries)
Record cash receipts: amount, date, received from, account, reference
Record cheque receipts: bank, cheque no., date, drawer, status (pending/cleared/bounced)
Cheque bounce handling: reverse receipt, notify user, add penalty charge
Bulk receipt entry for multiple customers in one session
Auto journal: Debit Cash/Bank → Credit Customer Account / Revenue
7.8.3 Payments (Credit Entries)
Record supplier payments: cash or cheque against purchase invoice
Record other payments: utilities, rent, maintenance, etc.
Payment approval workflow for amounts above configurable threshold
Cheque issuance register with status tracking
Auto journal: Debit Supplier Account / Expense → Credit Cash/Bank
7.8.4 Expenses
Expense categories: Utilities, Transport, Marketing, Maintenance, Other
Expense voucher: amount, date, category, description, payee, receipt attachment
Monthly expense summary by category
Recurring expense setup (e.g., monthly rent)
Expense approval workflow
7.8.5 Salaries
Employee register: name, role, base salary, allowances, deductions
Monthly payroll run: calculate net pay (gross - deductions - tax)
Payroll approval before disbursement
Salary slip generation (PDF)
PAYE / income tax calculation (configurable tax brackets)
Auto journal: Debit Salary Expense → Credit Cash/Bank
7.8.6 Tax Management
VAT / GST / sales tax rate configuration per product category
Tax collected on sales invoices (output tax) tracking
Tax paid on purchase invoices (input tax) tracking
Net tax liability calculation (output - input)
Tax return period configuration (monthly, quarterly)
Tax report for filing: summarised by period
7.8.7 Petty Cash
Petty cash fund setup: float amount, custodian assignment
Petty cash voucher: amount, date, purpose, approved by
Petty cash balance tracking (float - disbursements)
Replenishment request when balance falls below threshold
Petty cash reconciliation report
7.8.8 Financial Reports
Trial Balance: all account debit and credit balances
Profit & Loss Statement (monthly / quarterly / annual)
Balance Sheet
Cash Flow Statement
General Ledger: all entries per account with running balance
Bank Reconciliation
Accounts Payable Aging (supplier balances)
Accounts Receivable Aging (customer balances)

10. API Design (Node.js / Next.js API Routes)
10.1 API Conventions
Base URL: /api/v1/
Authentication: Bearer JWT (Supabase JWT passed in Authorization header)
Response format: { success: boolean, data: any, meta: { pagination }, error: string }
Pagination: ?page=1&limit=20 on all list endpoints
Filtering: ?filter[status]=active&filter[supplier_id]=uuid
Sorting: ?sort=created_at&order=desc

12. Security & Compliance
All API routes protected by Supabase JWT verification middleware
Row Level Security (RLS) policies per table: users only see records in their scope
Role-based permissions matrix: defined per module × action (Create / Read / Update / Delete / Approve)
All passwords hashed via Supabase Auth (bcrypt)
MFA enforced for Accountant, Management, and Super Admin roles
Sensitive data (salary, bank details) encrypted at column level
Audit log table: every INSERT/UPDATE/DELETE logged via PostgreSQL triggers
HTTPS enforced; HSTS header; CSP headers configured
Rate limiting on all API endpoints via Next.js middleware
Data retention policy: financial records retained minimum 7 years

14. Integrations (Phase 1 & 2)

Supabase Auth → All modules (JWT, RLS, user management)
