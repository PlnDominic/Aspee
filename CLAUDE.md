# CLAUDE.md ‚Äî Aspee Pharma ERP

## Project Overview

Aspee Pharma ERP is a full-featured pharmaceutical Enterprise Resource Planning system built for Aspee Pharmaceuticals Ltd (Ghana). It digitises and connects every department ‚Äî purchasing, stores, production, QA, sales, accounting, HR, and compliance ‚Äî into a single web application. The system is used internally by staff across roles, with strict role-based access controlling what each user can see and do.

---

## Tech Stack

- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript 5.x
- **UI Library:** React 19
- **Styling:** Vanilla CSS + CSS Variables (`globals.css`) ‚Äî no Tailwind
- **Icons:** Lucide React
- **Charts:** Recharts
- **State / Data Fetching:** TanStack React Query v5
- **Forms + Validation:** React Hook Form + Zod
- **Database / Auth / Realtime:** Supabase (PostgreSQL + Auth + Realtime + Storage)
- **Email:** Nodemailer (SMTP)
- **PDF Generation:** jsPDF + html2canvas
- **Deployment:** Vercel (`npx vercel --prod`)
- **Package Manager:** npm
- **Currency:** Ghana Cedis ‚Äî always format as `GH‚Çµ` using `formatCurrency()` from `src/lib/formatCurrency.ts`

---

## Coding Conventions

- **Components:** PascalCase filenames and function names; use `export default function ComponentName`
- **Hooks / utilities:** camelCase (`useFetch`, `useSave`, `formatCurrency`)
- **Data fetching:** use the custom React Query hooks in `src/lib/hooks.ts` (`useFetch`, `useSave`, `useAction`) ‚Äî do not write raw `useEffect` + `fetch` patterns
- **Forms:** React Hook Form with Zod resolver; define schemas in `src/lib/schemas.ts`
- **Modals:** each entity gets its own `*Modal.tsx` ‚Äî do not combine unrelated entity forms
- **Printable documents:** separate `Printable*.tsx` components; PDF is generated by `generatePDF()` from `src/lib/pdfGenerator.ts`
- **Styling:** write CSS in `globals.css` using CSS variables defined in `:root` / `.dark`; do not use inline styles or Tailwind
- **Types:** `any` is acceptable in modal and page components for Supabase rows ‚Äî do not tighten types in unrelated edits
- **Comments:** only add a comment when the WHY is non-obvious; no doc-block comments
- **Toasts:** use `toast()` from `sonner` for all user feedback

## Never Do This

- **Never install packages without asking** ‚Äî confirm with the user before `npm install`
- **Never rewrite files not explicitly requested** ‚Äî if a task touches `InvoiceModal.tsx`, leave `CustomerModal.tsx` alone
- **Never use Tailwind utility classes** ‚Äî this project uses vanilla CSS with CSS variables
- **Never hard-code currency symbols** ‚Äî use `formatCurrency()` and `GH‚Çµ`
- **Never skip the GitNexus impact check** ‚Äî see the GitNexus section below; run `gitnexus_impact` before editing any function/class
- **Never commit without running `gitnexus_detect_changes()`** ‚Äî required by project protocol
- **Never use `find-and-replace` for symbol renames** ‚Äî use `gitnexus_rename` instead
- **Never skip the Vercel deploy** ‚Äî run `npx vercel --prod` from `aspee-pharma/` after every completed task

## Important Context

- **Supabase RLS is active on all tables.** Schema changes require matching RLS policy migrations. Always write a `.sql` migration file and a corresponding `apply_*.js` runner for any DB change.
- **No ORM.** All queries use the Supabase JS client directly (`supabase.from('table').select()`).
- **Role list:** Super Admin, Sales Manager, Van Sales Rep, Purchasing Manager, Store Manager, Production Manager, Quality Assurance, Accountant, Internal Auditor, HR Manager, Managing Director. Route guards live in `src/middleware.ts`.
- **Realtime** is used for notifications (`useNotifications` in `hooks.ts`). Do not remove Supabase channel subscriptions without checking this hook.
- **PDF generation** renders a `Printable*` component to DOM then captures it with html2canvas. Keep printable components self-contained and avoid dynamic imports that delay render.
- **Email** goes through Nodemailer on the server (`/api/send-email`). SMTP credentials are in env vars ‚Äî never hard-code them.
- **All monetary values** are stored in GHS and displayed with `formatCurrency()`. Do not introduce a second currency or conversion layer.
